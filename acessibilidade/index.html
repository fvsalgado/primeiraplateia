Acessibilidade

Declaração: Março 2026 · Norma: WCAG 2.1 Nível AA

A acessibilidade não é uma funcionalidade extra — é uma exigência de base. Este site foi construído para ser utilizável por toda a gente, incluindo quem usa tecnologias de apoio.

Compromisso

A Primeira Plateia compromete-se a garantir a acessibilidade digital para pessoas com deficiência. Trabalhamos continuamente para melhorar a experiência de todos os utilizadores e aplicamos as normas de acessibilidade relevantes.

Norma de referência

Este site tem como referência as Web Content Accessibility Guidelines (WCAG) 2.1, Nível AA, publicadas pelo W3C. Estas normas definem como tornar o conteúdo web mais acessível a pessoas com deficiências visuais, auditivas, motoras ou cognitivas.

Estado de conformidade

A Primeira Plateia encontra-se em conformidade parcial com as WCAG 2.1 Nível AA. As áreas em que a conformidade é total ou parcial estão descritas abaixo.

Área Estado Notas
Navegação por teclado ✓ Conforme Todos os elementos interactivos são acessíveis por teclado. Foco visível em todos os estados.
Skip link ✓ Conforme "Saltar para o conteúdo principal" presente em todas as páginas.
Contraste de cor ✓ Conforme Todos os rácios de contraste verificados: mínimo 4.5:1 para texto normal, 3:1 para texto grande. Modos claro e escuro.
Estrutura semântica ✓ Conforme Uso correcto de headings (h1–h3), landmarks (nav, main, footer), listas e tabelas.
ARIA labels ✓ Conforme Todos os elementos interactivos têm labels descritivas. Regiões live para conteúdo dinâmico.
Texto alternativo em imagens ✓ Conforme Imagens decorativas com alt="" e aria-hidden. Imagens informativas com alt descritivo.
Preferência de movimento reduzido ✓ Conforme Todas as animações são desactivadas quando prefers-reduced-motion está activo.
Redimensionamento de texto ✓ Conforme O layout mantém-se funcional até 200% de zoom. Tipografia em unidades relativas (rem).
Modo escuro do sistema ✓ Conforme Detecta e respeita prefers-color-scheme automaticamente.
Conteúdo de terceiros (mapas, imagens) ◑ Parcial Os mapas OpenStreetMap e imagens dos teatros não estão sob nosso controlo. Os iframes de mapa têm title e aria-label.

Funcionalidades de acessibilidade

  • Skip link — O primeiro elemento de cada página é um link "Saltar para o conteúdo principal", activado com Tab ao carregar a página.
  • Foco visível — Todos os elementos interactivos têm um indicador de foco claro (contorno com offset) em modo de navegação por teclado.
  • Modo escuro — O site detecta automaticamente a preferência do sistema operativo e adapta a paleta. A preferência pode ser alterada manualmente com o botão na navegação.
  • Texto escalável — Toda a tipografia usa unidades relativas. O layout não quebra até 200% de zoom.
  • Animações opcionais — Quem tem prefers-reduced-motion: reduce activo não vê animações.
  • Regiões ARIA live — O conteúdo dinâmico (resultados de filtros, carregamento de espectáculos) é anunciado correctamente por leitores de ecrã.
  • Accordion acessível — As secções colapsáveis usam aria-expanded e aria-controls correctamente.

Limitações conhecidas

  • As imagens dos espectáculos são fornecidas pelos teatros e podem não ter texto alternativo descritivo disponível. Usamos alt="" para imagens de cartaz (conteúdo decorativo) para não gerar ruído nos leitores de ecrã.
  • Os mapas de localização dos teatros (OpenStreetMap) são de terceiros e podem ter limitações de acessibilidade fora do nosso controlo. Em alternativa, a morada textual está sempre disponível.

Tecnologias de apoio testadas

O site foi testado com as seguintes combinações:

  • VoiceOver (macOS + Safari)
  • NVDA (Windows + Firefox)
  • TalkBack (Android + Chrome)
  • Navegação por teclado (sem rato) em Chrome, Firefox e Safari
